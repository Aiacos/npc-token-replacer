const createMockPack = (packageName, label, docName = "Actor") => ({

Declare `let mockPacks = []` at module scope. In the top-level `beforeEach`:
1. Call `CompendiumManager.clearCache()` to clear all 4 private caches
2. Set `game.packs.filter = vi.fn(predicate => mockPacks.filter(predicate))` — CRITICAL: must execute the predicate, not return static data

**Test group 1: getCompendiumPriority (5 tests)**
- Known SRD package returns priority 1: `createMockPack("dnd5e", "SRD")` → 1
- Known Core package returns priority 2: `createMockPack("dnd-monster-manual", "MM")` → 2
- Known Expansion package returns priority 3: `createMockPack("dnd-forge-artificer", "Forge")` → 3
- Known Adventure package returns priority 4: `createMockPack("dnd-phandelver-below", "Phandelver")` → 4
- Unknown dnd- prefix defaults to priority 4: `createMockPack("dnd-unknown-adventure", "Unknown")` → 4
- Non-dnd package defaults to priority 1: `createMockPack("homebrew-pack", "Homebrew")` → 1

**Test group 2: detectWOTCCompendiums (4+ tests)**
Set `mockPacks` to various combinations, then call `CompendiumManager.detectWOTCCompendiums()`.
- Includes Actor compendiums with dnd- prefix: `createMockPack("dnd-monster-manual", "MM")` → included
- Includes Actor compendiums with dnd5e prefix: `createMockPack("dnd5e", "SRD Monsters")` → included
- Excludes non-Actor compendiums: `createMockPack("dnd-monster-manual", "MM Items", "Item")` → excluded
- Excludes non-WotC compendiums: `createMockPack("homebrew-monsters", "Homebrew")` → excluded
- Returns cached results on second call: call twice, verify `game.packs.filter` called only once (cache hit on second)

**Test group 3: getEnabledCompendiums (6+ tests)**
Set up `mockPacks` with 3 packs of different priorities for each test:
```javascript
const corePack = createMockPack("dnd-monster-manual", "Monster Manual");      // priority 2
const adventurePack = createMockPack("dnd-phandelver-below", "Phandelver");   // priority 4
const srdPack = createMockPack("dnd5e", "SRD Monsters");                      // priority 1
```

Tests:
- **"default" mode** includes only priority 1-2 (SRD + Core): `game.settings.get.mockReturnValue('["default"]')` → result contains corePack and srdPack, NOT adventurePack
- **"all" mode** includes every detected compendium: `game.settings.get.mockReturnValue('["all"]')` → all 3 packs
- **Specific pack IDs** filter to selected packs: `game.settings.get.mockReturnValue('["dnd-phandelver-below.monsters"]')` → only adventurePack
- **Corrupt JSON** falls back to default: `game.settings.get.mockImplementation(() => { throw new Error("Invalid JSON"); })` → same as default (corePack + srdPack)
- **Undefined setting** falls back to default: `game.settings.get.mockReturnValue(undefined)` → same as default
- **Empty array setting** falls back to default: `game.settings.get.mockReturnValue('[]')` → same as default

**Test group 4: clearCache (2 tests)**
- After clearCache, detectWOTCCompendiums re-reads game.packs: call detect → clearCache → change mockPacks → detect again → verify new packs returned
- After clearCache, getEnabledCompendiums re-reads settings: call getEnabled → clearCache → change settings mock → call again → verify new setting applied

**CRITICAL patterns from research:**
- `game.packs.filter = vi.fn(predicate => mockPacks.filter(predicate))` — the callback-execution mock is ESSENTIAL. A static mock would hide filtering bugs.
- `CompendiumManager.clearCache()` in EVERY `beforeEach` — 4 private caches must be cleared to prevent test pollution
- Do NOT test `loadMonsterIndex()` — it requires mocking `pack.getIndex()` and `pack.index.contents`, which is Phase 4 / integration territory per research decision
- Do NOT mock Logger
