Create `tests/lib/name-matcher.test.js` with the following test groups:

**Setup (top of file):**
```javascript
import { describe, it, expect, beforeEach, vi } from "vitest";
import { NameMatcher } from "../../scripts/lib/name-matcher.js";
```

Create a mock CompendiumManager object with `getCompendiumPriority` and `getIndexMap` as `vi.fn()`. In `beforeEach`, call `NameMatcher.setCompendiumManager(mockCompendiumManager)` and `.mockClear()` both mock functions.

**Test group 1: normalizeName (5+ tests)**
- Converts to lowercase and trims whitespace: `"Goblin Warrior"` → `"goblin warrior"`, `"  Dire Wolf  "` → `"dire wolf"`
- Removes special characters (Unicode-safe): `"Mind Flayer's Minion"` → `"mind flayers minion"`, `"Half-Dragon"` → `"halfdragon"`
- Normalizes internal whitespace: `"Death   Knight"` → `"death knight"`
- Handles null, undefined, empty string → `""`
- Handles numeric input gracefully (e.g., `123` should not throw)

**Test group 2: selectBestMatch (5+ tests)**
- Returns null for null and empty array input
- Returns single match directly when only one match exists
- Selects match from highest priority compendium (mock `getCompendiumPriority` to return different priorities for different packageNames). Use `"dnd5e"` → 1 and `"dnd-monster-manual"` → 2
- Uses pre-computed `.priority` field when available (set `NameMatcher.setCompendiumManager(null)` to test without CM)
- Breaks tie by collection name (alphabetical): both packs get same priority, pack with alphabetically-first collection wins

**Test group 3: findMatch — Stage 1 Exact Match (4+ tests)**
- Exact match found via array-scan path: set `mockCompendiumManager.getIndexMap.mockReturnValue(null)`, provide index array with matching normalizedName. Verify match returned.
- Exact match found via indexMap (O(1)) path: set `getIndexMap` to return a Map with the key. Verify match returned AND index array NOT scanned.
- No match returns null
- Empty/null creature name returns null

**Test group 4: findMatch — Stage 2 Variant Match (3+ tests)**
- Prefix stripping: `"Young Dragon"` matches index entry `"dragon"` (PREFIX_PATTERN strips "Young ")
- Suffix stripping: `"Goblin Warrior"` matches index entry `"goblin"` (SUFFIX_PATTERN strips " Warrior")
- Both prefix+suffix stripping: `"Elder Goblin Berserker"` matches index entry `"goblin"`
- Use `getIndexMap.mockReturnValue(null)` to force array-scan path for variant tests.

**Test group 5: findMatch — Stage 3 Partial Match (4 tests)**
- Match above 2/3 threshold: creature with 3 significant words, 2 matching → match found
- No match below threshold: only 1 of 3 significant words matches → null
- Short name below MIN_PARTIAL_LENGTH skips Stage 3: `"Rat"` (3 chars) → null even if partial words exist in index
- Bidirectional coverage check: entry's significant words must also be at least 50% covered

For all findMatch tests, construct inline mock index arrays with `{entry: {name}, pack: {metadata: {packageName, label}, collection}, normalizedName, significantWords, priority}` objects.

**IMPORTANT patterns from research to follow:**
- Never suppress Logger output — do NOT mock Logger
- Use `getIndexMap.mockReturnValue(null)` for array-scan tests, `getIndexMap.mockReturnValue(new Map(...))` for Map-path tests
- Construct index data inline (no factory functions)

Create `tests/lib/wildcard-resolver.test.js` with the following test groups:

**Setup (top of file):**
```javascript
import { describe, it, expect, beforeEach, vi } from "vitest";
import { WildcardResolver, DEFAULT_HTTP_TIMEOUT_MS } from "../../scripts/lib/wildcard-resolver.js";
```

In `beforeEach`: call `WildcardResolver.clearCache()` and `vi.restoreAllMocks()`.

**Test group 1: isWildcardPath (4 tests)**
- Returns true for paths containing `*`: `"tokens/specter-*.webp"` → true
- Returns false for normal paths: `"tokens/specter-1.webp"` → false
- Returns false for null, undefined, empty string
- Returns false for non-string input (e.g., `123`)

**Test group 2: selectVariant (6+ tests)**
Use `const variants = ["a.webp", "b.webp", "c.webp"]` for all mode tests.
- **none mode**: always returns first variant `"a.webp"`, nextIndex is 0
- **sequential mode**: index 0 → `"a.webp"`, index 1 → `"b.webp"`, index 2 → `"c.webp"`
- **sequential mode wraps**: index 3 → `"a.webp"` (modulo)
- **sequential nextIndex increments**: index 0 → nextIndex 1, index 5 → nextIndex 6
- **random mode with mocked Math.random**: `vi.spyOn(Math, "random").mockReturnValue(0.5)` → `"b.webp"` (Math.floor(0.5 * 3) = 1), nextIndex is 0
- **empty/null variants**: returns `{path: null, nextIndex: sequentialIndex}`

**Test group 3: resolveWildcardVariants (4+ tests)**
Mock `global.fetch` to simulate HEAD request responses.
- Discovers available variants: mock fetch so URLs containing `specter-1.webp` and `specter-2.webp` return `{ok: true}`, all others return `{ok: false}`. Verify result contains exactly those 2 paths.
- Caches results: call twice with same path, verify fetch called only during first call (second returns from cache)
- Cache cleared via clearCache(): resolve, clear, resolve again — fetch should be called both times
- Handles all failed requests gracefully: mock fetch to always return `{ok: false}` → returns empty array, does not throw

For fetch mock pattern, use default-reject approach:
```javascript
global.fetch = vi.fn((url) => {
  if (url.includes("specter-1.webp")) return Promise.resolve({ ok: true });
  return Promise.resolve({ ok: false, status: 404 });
});
```

**Test group 4: resolve (3 tests)**
- Returns resolved path when variants exist: mock `resolveWildcardVariants` via fetch mock to find variants
- Falls back to fallbackPath when no variants found: mock fetch to always 404, provide a non-wildcard fallbackPath → verify it's returned
- Falls back to mystery-man when no variants and no valid fallback: mock fetch to always 404, no fallback → returns `"icons/svg/mystery-man.svg"`

**IMPORTANT patterns from research:**
- `unstubGlobals: true` in vitest.config.js auto-restores `global.fetch` between tests — do NOT manually restore
- `vi.restoreAllMocks()` in beforeEach handles `Math.random` spy cleanup
- Do NOT mock Logger
