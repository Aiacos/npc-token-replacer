## Directory Purposes

**scripts/:**
- Purpose: Module entry point and all logic
- Contains: Single ES module file with 8 OOP classes
- Key files: `main.js`
- Notes: No external dependencies; plain ES6 modules

**lang/:**
- Purpose: User-facing string localization
- Contains: JSON file with i18n keys organized by feature area
- Key files: `en.json` (English translations)
- Pattern: `NPC_REPLACER.<Category>.<Key>` e.g., `NPC_REPLACER.Button`, `NPC_REPLACER.Settings.VariationMode.Name`

**templates/:**
- Purpose: Handlebars UI templates
- Contains: Compendium selector form HTML with embedded styles and client-side script
- Key files: `compendium-selector.html`
- Notes: Uses Foundry's Handlebars templating for form generation

**releases/:**
- Purpose: Packaged distribution files
- Contains: ZIP archives with version-specific naming (`npc-token-replacer-vX.Y.Z.zip`)
- Notes: Auto-generated by build script; never edit directly

## Naming Conventions

**Files:**
- `main.js`: Single monolithic module file
- `compendium-selector.html`: Handlebars template following kebab-case
- `en.json`: Language code for localization

**Directories:**
- `scripts/`: Foundry convention for ES modules
- `lang/`: Foundry convention for localization files
- `templates/`: Foundry convention for Handlebars templates
- `releases/`: Release distribution folder

**Classes (in main.js):**
- `Logger`: Simple utility class (uppercase)
- `FolderManager`: Manager class (ManagerSuffix)
- `CompendiumManager`: Domain-specific manager (ManagerSuffix)
- `NameMatcher`: Utility class with static methods (NoSuffix)
- `TokenReplacer`: Domain-specific utility (NoSuffix)
- `WildcardResolver`: Domain-specific utility (ResolverSuffix)
- `NPCTokenReplacerController`: Facade controller (ControllerSuffix)
- `CompendiumSelectorForm`: FormApplication subclass (FormSuffix)

**Functions:**
- `registerSettings()`: camelCase
- `registerControlButton()`: camelCase
- `escapeHtml()`: camelCase

**Constants:**
- `MODULE_ID`: UPPER_CASE
- `DEFAULT_HTTP_TIMEOUT_MS`: UPPER_CASE
- Static readonly getters like `FOLDER_NAME`, `WOTC_MODULE_PREFIXES`: UPPER_CASE

**Private Fields:**
- `static #field`: ES6 private static field convention for encapsulation

## Where to Add New Code

**New Feature (e.g., token filtering logic):**
- Primary code: `scripts/main.js` - add new class or extend existing (e.g., new `TokenFilter` class near TokenReplacer)
- Controller integration: Add public method to `NPCTokenReplacerController` if user-facing
- Settings: Add to `registerSettings()` if configurable
- Localization: Add keys to `lang/en.json` with pattern `NPC_REPLACER.Settings.<FeatureName>.<Key>`
- Debug API: Add to `NPCTokenReplacerController.getDebugAPI()` if needs console access

**New Dialog/Form:**
- Implementation: Add FormApplication subclass in `scripts/main.js` after `CompendiumSelectorForm` (line 1914+)
- Template: Create new HTML file in `templates/` with Handlebars syntax
- Triggering: Call from controller method or hook handler
- Example pattern: See `CompendiumSelectorForm` (lines 1914-2005) for full example

**New Utility Function:**
- Shared helpers: Add as static method to appropriate utility class:
  - Name/text operations → `NameMatcher`
  - File/path operations → `WildcardResolver`
  - Folder operations → `FolderManager`
  - Logging → `Logger`
- Standalone functions (pure utilities): Add after class definitions before Hooks (see `escapeHtml` at line 2059)

**New Localization:**
- Location: `lang/en.json`
- Pattern: Mirror the category structure (`NPC_REPLACER.<Category>.<Key>`)
- Categories: `Title`, `Button`, `Settings`, `Messages` (see existing structure)
- Reference in code: Use `game.i18n.localize("NPC_REPLACER.Key")` or `game.i18n.format("NPC_REPLACER.Key", {data})`

**Adding a New Setting:**
- Define: Add registration in `registerSettings()` (lines 1870-1904)
- Scope: Use `scope: "world"` for GM-only settings
- Storage: For complex data, use JSON.stringify/JSON.parse for reliability
- UI: Create FormApplication subclass or use `config: true` for simple settings
- Usage: Access via `game.settings.get(MODULE_ID, "settingName")`

## Monolithic Design Rationale

All logic is intentionally in a single `scripts/main.js` file because:

1. **No Build System**: Foundry VTT runs modules as ES modules directly in browser
2. **Simple Deployment**: Single-file module means copying one file into Foundry's module directory
3. **Class Organization**: Logic is organized into 8 cohesive classes with single responsibilities
4. **Private Fields**: ES6 `#privateField` syntax provides encapsulation without needing separate files
5. **Minimal Overhead**: No bundler, no transpilation, plain modern JavaScript

## Imports and Dependencies

**External Dependencies:**
- Foundry VTT core APIs: `game`, `canvas`, `ui`, `Dialog`, `FormApplication`, `Folder`, `Hooks`
- System: dnd5e (required, specified in module.json)
- JavaScript: ES2022+ (Template literals, private fields, Array methods, Map, Set, AbortController)

**No npm dependencies in production** (only dev eslint for linting)

**Local Imports:**
- All classes defined in same file; accessed directly by reference
- No module imports needed since everything is in one file

## Build and Release Process

**Build Script** (`build.sh` / `build.bat`):
- Reads version and module ID from `module.json`
- Creates clean directory `releases/{id}-v{version}/`
- Copies all source files
- Generates `module.json` with correct download URL
- Creates ZIP: `releases/{id}-v{version}.zip`
- Cleans up temporary directory

**Manifest URL:**
- Foundry looks up at: `https://github.com/Aiacos/npc-token-replacer/releases/latest/download/module.json`
- Downloads ZIP from: URL in module.json `download` field

**Update Cycle:**
1. Bump version in `module.json` (both `version` and `download` URL fields)
2. Run `bash build.sh`
3. Commit module.json: `git commit -m "Bump version to X.Y.Z"`
4. Create GitHub release: `gh release create vX.Y.Z releases/npc-token-replacer-vX.Y.Z.zip module.json --title "vX.Y.Z - Description"`
